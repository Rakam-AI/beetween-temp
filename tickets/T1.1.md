# T1.1 - Rename Project Identity & Packages


**Depends on:** Nothing (first ticket)
**Blocks:** All subsequent tickets

**Context:**
The base template at `poc-beetween/` is a Kotlin Spring Boot hexagonal architecture project currently named `poc-btwgen-api` with root package `com.beetween.pocbtwGenapi`. It contains a sample "Greeting" domain. The project structure follows strict hexagonal architecture enforced by ArchUnit tests.

**Objective:**
Rename the entire project identity to match the Demand Approval Request (DAR) service naming without changing any logic or architecture.

**Instructions:**
1. In `pom.xml`:
   - Change `<artifactId>` from `poc-btwgen-api` to `dar-api`
   - Change `<name>` to `Demand Approval Request API`
   - Change `<description>` to `API for managing recruitment approval requests and validation workflows`
   - Keep the parent `com.beetween:springboot-parent:1.7.0` unchanged

2. Rename the Kotlin root package from `com.beetween.pocbtwGenapi` to `com.beetween.dar`:
   - Rename all directories under `src/main/kotlin/` and `src/test/kotlin/`
   - Update all `package` declarations in every `.kt` file
   - Update all `import` statements referencing the old package

3. In `application.yml`:
   - Update `spring.application.name` to `dar-api`
   - Update Keycloak `client-id` to `dar-api`

4. In `Dockerfile`: Update the JAR filename reference if it uses the artifact name

5. In `docker-compose.yml`: Rename the service from any poc reference to `dar-api`

6. In `src/main/kotlin/.../Application.kt`: Ensure the main class package is `com.beetween.dar`

7. Update `README.md` header to reference the new project name

**Acceptance Criteria:**
- Project compiles with `./mvnw clean compile`
- All package declarations use `com.beetween.dar`
- No references to `pocbtwGenapi` or `poc-btwgen-api` remain in the codebase
- Docker build still works
