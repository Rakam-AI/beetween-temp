# T1.7 - Set Up Project Package Structure


**Depends on:** T1.2
**Blocks:** T2.1, T2.2

**Context:**
After removing the Greeting sample, we need to establish the target package structure as defined in the API specification. This creates all necessary directories and placeholder files to guide development.

Reference: `API_Technical Specification V1.md` > Contraintes techniques > Structure du projet

**Objective:**
Create the complete package structure for the DAR service following hexagonal architecture, with appropriate placeholder marker files.

**Instructions:**
Create the following directory/package structure under `src/main/kotlin/com/beetween/dar/`:

```
domain/
├── request/
│   ├── model/          # Request aggregate, value objects
│   ├── event/          # Domain events (RequestCreated, etc.)
│   └── error/          # Domain errors
├── workflow/
│   ├── model/          # Workflow aggregate, Step, AdditionalField
│   ├── event/          # Workflow domain events
│   └── error/          # Workflow domain errors
└── shared/
    └── model/          # Shared value objects (ActorReference, etc.)

application/
├── request/
│   ├── port/
│   │   ├── input/      # Use case interfaces (driving ports)
│   │   └── output/     # Repository/publisher interfaces (driven ports)
│   └── usecase/        # Use case implementations
├── workflow/
│   ├── port/
│   │   ├── input/
│   │   └── output/
│   └── usecase/
└── shared/
    └── port/
        └── output/     # Shared ports (EventPublisherPort, etc.)

adapter/
├── input/
│   └── web/
│       ├── request/
│       │   └── dto/    # Request DTOs
│       ├── workflow/
│       │   └── dto/    # Workflow DTOs
│       ├── dev/        # Dev-only endpoints (token)
│       └── errormanager/  # Already exists - keep
└── output/
    ├── persistence/
    │   ├── request/
    │   │   └── entity/ # MongoDB entities
    │   └── workflow/
    │       └── entity/
    └── messaging/      # Pulsar adapter

config/                 # Already exists - keep structure
```

For each new package, create a minimal `.gitkeep` or a meaningful placeholder:
- For `port/input/` directories: create a placeholder interface file with a TODO comment
- For `port/output/` directories: create a placeholder interface file with a TODO comment
- This ensures the directory structure is committed to git

Also create `src/test/kotlin/com/beetween/dar/` mirroring the same top-level structure for tests.

**Acceptance Criteria:**
- Full package structure exists and is committed
- Project still compiles
- ArchUnit tests are updated to reference new packages (they may be relaxed/skipped for now)
